🚀 DocuTranslate 断点续传加强版（Hash 隔离机制）
本版本为 DocuTranslate 引入了**“零侵入、跨重启、无感恢复”**的纯本地断点续传系统，彻底解决翻译长文本（如大部头 EPUB、长篇 PDF）时因大模型 API 频率限制（429）、超时（524）或网络波动导致的进度全部丢失问题。

✨ 核心特性

🔄 跨会话与跨重启恢复
哪怕你的网页关了、Docker 容器重启了，甚至是服务器断电了，只要 output 目录还在，重新上传文件就能瞬间接上进度。

🔒 基于内容的绝对绑定 (Hash Isolation)
抛弃传统易丢失的任务 ID，系统会在底层对你要翻译的原文内容提取 MD5 专属指纹。这意味着不管前端页面怎么刷新，只要你传的是同一本书，进度就绝对不会丢。

🗑️ 阅后即焚 (Auto-Cleanup)
当一本书 100% 翻译成功并拼装完毕后，系统会自动销毁该书专属的 .jsonl 断点缓存文件，绝不占用服务器硬盘空间。

🛡️ 拒绝残缺脏数据
严格遵守数据原子性。如果重试耗尽后大模型只返回了“半截”译文，系统会直接丢弃并标记为失败，防止破坏 EPUB/PDF 等复杂文档的闭合标签导致生成乱码。

### 🛠️ 如何测试我的修改版本？

你可以直接克隆我的 Fork 仓库进行测试。为了防止和旧版本冲突，请按照以下步骤全新部署：

**1. 彻底清理旧环境与数据（注意备份成品）**
```bash
docker stop docutranslate && docker rm docutranslate
rm -rf ~/docker_sources/docutranslate/output/*
```

**2. 拉取我的修改版仓库并强制构建**
```bash
# 克隆包含断点续传功能的 Fork 仓库
git clone https://github.com/wu1552944668/docutranslate.git
cd docutranslate

# 强制无缓存构建
docker build --no-cache -t docutranslate:latest .
```

**3. 启动并映射输出目录**
```bash
mkdir -p ~/docker_sources/docutranslate/output
chmod 777 ~/docker_sources/docutranslate/output

docker run -d \
  --name docutranslate \
  --restart=always \
  -p 8010:8010 \
  -v ~/docker_sources/docutranslate/output:/app/output \
  docutranslate:latest
```

**测试方法：**
设置一个极低的超时时间或速率限制来触发任务失败。任务停止后，直接在网页上再翻译一次同一个文件，观察后台日志打印的 `已恢复 XX 条匹配当前任务的记录` 即可验证。
